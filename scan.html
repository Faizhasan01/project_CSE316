About

SCAN

he SCAN disk scheduling algorithm is a method used by the operating system to manage disk read/write requests efficiently. It moves the disk head from one end of the disk to the other, servicing requests along the way, then reverses direction and repeats the process.

This movement pattern resembles an elevator, which is why SCAN is also called the "Elevator Algorithm."


Advantages

  1. Low variance of response time.

  2. High throughput.

  3. Average response time.

  4. No starvation.

Disadvantages

  1. Long waiting time for recently visited cylinders.

  2. Head moves to the end of the disk even if no requests are there.

Algorithm

  1. Request array stores requested track indexes in ascending order.

  2. Direction represents head movement (left or right).

  3. Service all tracks in the current direction.

  4. Calculate absolute distance of the track from the head.

  5. Increment seek count.

  6. Set serviced track as the new head position.

  7. Repeat until reaching disk end.

  8. Reverse direction and repeat until all tracks are serviced.
